New Metric Proposal

CVS — Compounding Velocity Score

The existing RDS metric captures efficiency at a snapshot. CVS captures the trajectory — how much a system's accuracy grows as reasoning depth increases. A system with high CVS doesn't just answer better; it gets better as the problem gets harder.

Compounding Velocity Score (CVS)
CVS = Σ(ΔF1_h) × MonotonicRate

Where ΔF1_h = F1(hop_h) − F1(hop_h−1), and MonotonicRate = fraction of positive ΔF1 steps.

CVS captures both the magnitude of compounding gain and its reliability. A system that improves 40% across 3 hops but degrades on 2 others is penalized relative to one that improves consistently.

GraphRAG Distinction — Clarified

The reviewer correctly notes: Microsoft's GraphRAG uses community detection + summarization, not deterministic DAG traversal. CKG is reasoning-native — the graph structure IS the retrieval path, not a preprocessing step. This distinction is why CKG compounds at depth while GraphRAG plateaus.

🏗

The Moat Question — Building at Scale

The reviewer identified the production bottleneck: "how automated is extraction for messy enterprise data?" Track 2 answers this directly — the GLP-1 domain was built from ClinicalTrials.gov API in a single session, no expert curation, achieving F1 = 0.530. Structured sources eliminate the extraction problem. SEC EDGAR, USPTO, clinical APIs: structure is already latent.

Research Agenda — Open Questions

Three Questions That Define the Next Phase

These are the reviewer's open questions, restated as falsifiable research directions.

  • DAG authoring at scale: Can Track 2 methodology (structured API sources → CKG in one session) generalize to 10, 50, 100 enterprise verticals? Hypothesis: yes, with pipeline automation. The bottleneck is source availability, not extraction cost.
  • Frontier model injection: Does injecting CKG as structured context into GPT-5, Gemini Ultra, or Claude Opus narrow the F1 gap — or does the structural traversal advantage hold independent of model capability? This is a direct test of whether CKG's value is in the structure or the prompting.
  • Commercial pilot F1 floor: What is the minimum acceptable F1 for enterprise adoption in regulated verticals (healthcare, pharma, financial)? If Track 2 achieves 0.530 without curation, what does curated Track 1 achieve on the same queries? That delta defines the curation premium.
Proposed Next Experiment

ΔRDS/ΔDomains — Multi-Graph Collision Velocity

The existing benchmark measures a single domain at a time. The compounding velocity hypothesis predicts that when multiple domain graphs are available simultaneously, RDS grows superlinearly — because bridge nodes emerge that no single graph could produce. This is the direct test of Patent Claim 1.

Experiment Design

Does RDS compound as domain count increases?

Run 7,928 existing queries against CKG systems at 1, 5, 10, 20, and 45 domains simultaneously. Measure RDS at each interval. If the curve is superlinear, that is direct empirical evidence of emergent intelligence from multi-graph collision.

  • Step 1: Baseline — run T5 (cross-concept) queries against 1 domain only. Record RDS.
  • Step 2: 5 domains — run same T5 queries with 5 domains loaded simultaneously. Record RDS.
  • Step 3: 10, 20, 45 domains — repeat. Plot RDS vs domain count.
  • Step 4: If curve is superlinear (power law), compute the exponent — that is the compounding velocity coefficient.
  • Step 5: The bridge node count (concepts that appear in 2+ domain graphs) should grow at the same rate as RDS. That's the mechanism proof.
  • Hypothesis: RDS ∝ D^α where D = domain count and α > 1 (superlinear) — vs RAG where α ≈ 0 (doesn't scale).
